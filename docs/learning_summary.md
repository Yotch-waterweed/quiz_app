# 今日の学習まとめ: Web開発の基本

## 1. ローカル環境でJavaScriptが動かない問題

### 現象
HTML、CSS、JavaScriptで作成したWebページをローカルファイルとしてブラウザで開いた際、画面が真っ白になり、JavaScriptで実装したクイズが表示されなかった。

### 原因
**ブラウザの「同一オリジンポリシー」というセキュリティ機能。**
- Webページ (`index.html`) が、自分のPCのファイルシステム (`file://`) から読み込まれている。
- そのページ内のJavaScriptが、別のファイル (`quiz.json`) を読み込もうとした。
- ブラウザはこれを「異なる場所からのリソース要求」とみなし、セキュリティリスクを避けるために通信をブロックした。

## 2. 解決策：ローカルWebサーバーの起動

この問題を解決するために、自分のPC上で一時的なWebサーバーを起動した。

### 方法
ターミナルでプロジェクトのディレクトリに移動し、以下のコマンドを実行。
```bash
python3 -m http.server
```

### なぜ解決したか
- このコマンドで、PC内に`http`プロトコルで通信するWebサーバーが起動する。
- `http://localhost:8000` というWebアドレスにアクセスすると、HTMLファイルもJSONファイルも同じサーバー（同じオリジン）から提供されることになる。
- その結果、ブラウザは通信をブロックせず、JavaScriptが正常に `quiz.json` を読み込めるようになった。

## 3. Webサイトの公開方法
作成したWebアプリをインターネット上に公開するには、Webサーバーにファイルを配置する必要がある。

- **レンタルサーバーを借りる:** 従来からの一般的な方法。
- **静的サイトホスティングサービスを利用する:** 今回作成したような、バックエンド処理の不要な「静的サイト」の場合、こちらが手軽で主流。
    - **代表的なサービス:**
        - GitHub Pages (無料)
        - Netlify (無料プランあり)
        - Vercel (無料プランあり)
    - これらのサービスを使えば、無料で簡単にWebサイトを公開できる。
