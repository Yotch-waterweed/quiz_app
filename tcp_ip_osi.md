# OSI参照モデルとTCP/IPモデルの概要

## OSI参照モデル (Open Systems Interconnection Reference Model)

OSI参照モデルは、国際標準化機構(ISO)によって策定された、コンピュータネットワークにおける通信機能を7つの階層（レイヤー）に分割したモデルです。各レイヤーは特定の役割を持ち、下位のレイヤーの機能を利用し、上位のレイヤーに機能を提供します。

- **第7層: アプリケーション層**
  - ユーザーが直接操作するアプリケーションに特有のプロトコルを提供します。
  - 例: HTTP (Web閲覧), FTP (ファイル転送), SMTP (メール送信)

- **第6層: プレゼンテーション層**
  - データの表現形式を変換します。文字コードの変換やデータの暗号化、圧縮などを行います。
  - 例: JPEG, GIF, TLS/SSL

- **第5層: セッション層**
  - 通信の開始、維持、終了（セッション）を管理します。
  - 例: NetBIOS

- **第4層: トランスポート層**
  - エンドツーエンドの信頼性の高いデータ転送を提供します。データが正しく相手に届いたかを確認し、エラー訂正や再送制御を行います。
  - 例: TCP, UDP

- **第3層: ネットワーク層**
  - 異なるネットワーク間の通信を可能にし、宛先までの最適な経路を選択します（ルーティング）。
  - 例: IP (インターネットプロトコル)

- **第2層: データリンク層**
  - 同一ネットワーク内の直接接続された機器間でのデータ転送を行います。
  - 例: Ethernet, Wi-Fi

- **第1層: 物理層**
  - データを電気信号や光信号に変換し、物理的な媒体（ケーブル、光ファイバーなど）を通じて伝送します。

## TCP/IPモデル (Transmission Control Protocol/Internet Protocol)

TCP/IPは、現在のインターネットで事実上の標準（デファクトスタンダード）として利用されているプロトコル群を4つの階層にモデル化したものです。OSI参照モデルよりも実践的で、よりシンプルな構造をしています。

- **第4層: アプリケーション層**
  - OSIモデルのアプリケーション層、プレゼンテーション層、セッション層の機能を包含します。
  - 例: HTTP, FTP, SMTP, DNS

- **第3層: トランスポート層**
  - OSIモデルのトランスポート層に相当します。
  - 例: TCP (信頼性重視), UDP (速度重視)

- **第2層: インターネット層**
  - OSIモデルのネットワーク層に相当します。
  - 例: IP

- **第1st層: ネットワークインターフェース層**
  - OSIモデルのデータリンク層と物理層を合わせたものです。
  - 例: Ethernet, Wi-Fi

## 関係性と違い

- **目的**: OSIモデルは「理想的なネットワークはどうあるべきか」という指針を示す「参照モデル」であるのに対し、TCP/IPはインターネットの発展と共に実際に使われてきたプロトコルを体系化した「実践的なモデル」です。
- **階層**: OSIは7階層、TCP/IPは4階層です。TCP/IPはOSIの上位3層（アプリケーション、プレゼンテーション、セッション）を1つのアプリケーション層に、下位2層（データリンク、物理）を1つのネットワークインターフェース層にまとめています。
- **普及度**: OSIプロトコル自体は広く普及しませんでしたが、その階層化の考え方はネットワーク技術を理解し、議論するための共通言語として非常に重要です。一方、TCP/IPは現在のインターネットの基盤となっています。
